// SELF-HEALING & RELIABILITY

> СУТЬ:
УПАЛ — ОТЖАЛСЯ.
Способность агента самостоятельно исправлять свои ошибки,
не беспокоя человека по пустякам.

> ПОЧЕМУ ЭТО ВАЖНО (WHY IT MATTERS):
 1. Стабильность: В реальном мире API падают, а данные бывают кривыми.
 2. Автономность: Агент, который зовет "маму" при каждой ошибке —
    бесполезен.
 3. Обучение: Исправляя ошибки, агент лучше понимает контекст задачи.

---

**ПОГРУЖЕНИЕ:**

Главное отличие Junior-разработчика от Senior — это реакция на ошибку. Junior паникует. Senior читает StackTrace.
Агенты умеют быть сеньорами.

Когда агент пишет код на Python и получает ошибку `ModuleNotFoundError`, он не должен падать. Он должен:
1.  Прочитать ошибку.
2.  Понять: "Ага, я забыл установить библиотеку `pandas`".
3.  Выполнить `pip install pandas`.
4.  Повторить запуск кода.

Этот цикл называется **Reflection (Рефлексия)**.

```text
    ПОПЫТКА №1
    +-----------------------+
    | Code: import numpy    |
    | Status: CRASH!        | <--- ОШИБКА
    +-----------+-----------+
                |
                v
    +-----------------------+
    | ANALYZER (LLM)        |
    | "Ошибка: модуль не    |
    |  найден. План:        |
    |  установить numpy."   |
    +-----------+-----------+
                |
                v
    +-----------------------+
    | ACTION: pip install   |
    +-----------+-----------+
                |
                v
    ПОПЫТКА №2
    +-----------------------+
    | Code: import numpy    |
    | Status: SUCCESS!      | <--- УСПЕХ
    +-----------------------+
```

**Правило надежности:**
Всегда давайте агенту шанс исправиться. Установите лимит попыток (например, 3), чтобы он не зациклился, пытаясь пробить стену головой. Но никогда не прерывайте выполнение после первой неудачи.
