// TOOLS & SKILLS

> СУТЬ:
РУКИ ДЛЯ НЕЙРОСЕТИ.
Tools (Инструменты) — это функции, которые модель может
"вызвать", если поймет, что не справляется сама.

> ПОЧЕМУ ЭТО ВАЖНО (WHY IT MATTERS):
 1. Актуальность: Модель не знает курс доллара сейчас. API знает.
 2. Точность: Модель плоха в математике (2+2=?). Калькулятор идеален.
 3. Безопасность: Мы даем доступ только к конкретным функциям,
    а не ко всему серверу.

---

**ПОГРУЖЕНИЕ:**

Как это работает "под капотом"? Магия называется **Function Calling**.

1.  Вы описываете инструменту (например, "Погода") в формате JSON:
    *"Функция `get_weather` принимает `city` (строка)"*.
2.  Вы посылаете это описание вместе с вопросом пользователя в LLM.
3.  LLM видит вопрос "Как там в Лондоне?" и вместо ответа генерирует JSON:
    `{ "function": "get_weather", "args": { "city": "London" } }`
4.  Ваш код выполняет эту функцию, получает "+15C, Дождь" и возвращает это обратно в LLM.
5.  LLM, наконец, отвечает юзеру: "В Лондоне сейчас +15 и дождь".

Для модели инструмент — это просто возможность делегировать задачу.

```text
    USER: "Посчитай 349 * 123"
           |
           v
    +-------------+
    |     LLM     |  ---> "Языковая модель, я гуманитарий..."
    | (Thinking)  |  ---> "Но у меня есть инструмент Calc!"
    +-------------+
           |
           | JSON: { "call": "calc", "args": "349*123" }
           v
    +-------------+
    | CALCULATOR  |  <--- (Выполнение Python кода)
    +-------------+
           |
           | Return: 42927
           v
    +-------------+
    |     LLM     |
    +-------------+
           |
           v
    AGENT: "Результат: 42 927"
```

**Ключевой момент:**
Модель *не нажимает кнопки*. Она генерирует *текст* (JSON), который ваш бэкенд превращает в нажатие кнопок. Это иллюзия действий, но очень убедительная.
