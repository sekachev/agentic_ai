```text
Промпт — это не "просьба". Это исполняемый код, написанный

> ПОЧЕМУ ЭТО ВАЖНО (WHY IT MATTERS):
 1. Точность: В коде (Python) ошибка в синтаксисе ломает
    программу. В промпте ошибка в смысле ломает результат.
 2. Детерминизм: Хороший промпт дает предсказуемый
    результат. Плохой промпт — это лотерея.
 3. Инженерия: Промпт нужно версионировать, тестировать
    и оптимизировать, как любой другой софт.
```

#### [Правая страница]

**ПОГРУЖЕНИЕ:**

Раньше, чтобы заставить компьютер нарисовать круг, вы писали:
`ctx.arc(100, 75, 50, 0, 2 * Math.PI);`

Теперь вы пишете:
`"Draw a perfect circle."`

Произошла абстракция. Мы поднялись на уровень выше.
Но механика осталась прежней. Ваши слова проходят через компилятор (Токенизатор -> Модель -> Декодер) и превращаются в действия.

**Естественный язык (Natural Language) — это API к реальности.**
Если вы говорите нечетко, вы получаете нечеткий результат.
Если вы говорите программисту "Сделай красиво", он сделает как умеет.
Если вы говорите ИИ "Напиши пост", он напишет спам.

Промпт-инжиниринг — это умение переводить *намерение* (Intention) в *инструкцию* (Instruction) так, чтобы машина не могла вас неправильно понять.
Это искусство быть занудой.
Вы должны объяснить машине контекст, роль, формат вывода и ограничения.

> **Метафора:**
> Промпт — это заклинание.
> Одно неверное слово — и вместо золота вы получаете черепки.
> Волшебников (разработчиков) сменили Заклинатели (операторы промптов).

```text
       CODE vs PROMPT
       
       [ Python ]
       def greet(name):
           return f"Hello, {name}"
           
       [ Prompt ]
       "You are a polite greeter.
        When given a name, return ONLY a greeting.
        No yapping."
       
       RESULT:
       Both are instructions.
       One is rigid (Syntax).
       One is fluid (Semantics).
```
