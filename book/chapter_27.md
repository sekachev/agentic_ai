// MODEL CONTEXT PROTOCOL (MCP)

> СУТЬ:
USB-ПОРТ ДЛЯ ИИ.
Единый стандарт, позволяющий подключать любые данные
к любым агентам без переписывания кода.

> ПОЧЕМУ ЭТО ВАЖНО (WHY IT MATTERS):
 1. Унификация: Один коннектор для базы данных работает
    и в Claude Desktop, и в Cursor, и в вашем скрипте.
 2. Экосистема: Не нужно писать интеграции с Slack/GitHub
    с нуля. Просто возьмите готовый MCP-сервер.
 3. Безопасность: Локальный запуск. Ключи остаются у вас.

---

**ПОГРУЖЕНИЕ:**

Раньше каждый разработчик писал свой "велосипед", чтобы подключить Google Drive к ChatGPT. Это создавало хаос.
Anthropic предложила стандарт **MCP**. Это как USB. Неважно, какая у вас мышка и какой компьютер. Если есть USB — они заработают.

MCP делит мир на:
*   **Хосты (Clients):** Приложения, где живет ИИ (Claude Desktop, IDE).
*   **Серверы (Servers):** Небольшие программы, дающие доступ к данным (SQLite, Filesystem, Slack).

Через MCP сервер передает три типа объектов:
1.  **Resources:** Данные для чтения (файлы, логи).
2.  **Tools:** Функции для исполнения (выполнить SQL, отправить сообщение).
3.  **Prompts:** Шаблоны команд.

```text
       HOST (CLIENT)              MCP SERVERS
    +----------------+          +-------------+
    |                | <------> | GitHub MCP  |
    | Claude Desktop |          +-------------+
    |                |
    | (Ваш Агент)    |          +-------------+
    |                | <------> | Postgres MCP|
    +----------------+          +-------------+
                                      ^
                                      |
                                  SQL Query
                                      |
                                +------------+
                                |  Real DB   |
                                +------------+
```

**Эффект:**
Вы запускаете локально сервер `npx -y @modelcontextprotocol/server-filesystem` и ваш Claude внезапно "видит" все ваши локальные файлы. Без загрузки в облако. Это меняет правила игры для приватности и скорости разработки.
