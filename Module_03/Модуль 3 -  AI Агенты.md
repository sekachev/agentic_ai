

### Модуль 3 - AI Агенты (4 часа)

#### Блок 1: Анатомия Агента — "Двигатель и Автомобиль" (1 час)
*Метафора: LLM — это мощный двигатель, но сам по себе он никуда не едет. Ему нужен автомобиль.*

1. **Двигатель (LLM)**: Отвечает за reasoning (мышление) и генерацию.
2. **Автомобиль (Обвязка/Фреймворк)**:
   - **Инструменты (Tools/Actuator)**: Руки агента. API, поиск в интернете, калькулятор, доступ к файловой системе.
   - **Память (Memory)**:
     - Краткосрочная (Context window).
     - Долгосрочная (RAG, Векторные БД, История записей).
   - **Восприятие (Sensors)**: Входные данные (Текст, Аудио, Картинки).
3. **Интерфейс (Dashboard)**:
   - Почему чат — не всегда лучший интерфейс?
   - Концепция Generative UI (интерфейс, который подстраивается под контекст).
   - Human-in-the-loop: место человека в управлении "автомобилем".

#### Блок 2: Паттерны проектирования агентов (1 час)
*Как заставить модель "думать" перед действием?*

1. **Базовые циклы**:
   - **ReAct** (Reason + Act): Подумал -> Сделал -> Посмотрел результат -> Повторил.
2. **Стратегии планирования**:
   - **Chain of Thought**: "Давай думать шаг за шагом".
   - **Planning**: Сначала план, потом исполнение.
   - **Reflection**: Критика собственных действий ("Правильно ли я сделал?").
3. **Архитектуры**:
   - Одиночные агенты vs Мульти-агентные системы (Orchestrator-Workers).

#### Блок 3: No-code подход (n8n) + Практика (1 час)
*Быстрый старт и прототипирование.*

1. **n8n как среда для агентов**:
   - AI Agent Node в n8n.
   - Подключение инструментов (LangChain tools внутри n8n).
2. **Сравнение**:
   - Плюсы: Визуально, быстро, низкий порог входа.
   - Минусы: Сложно дебажить сложные циклы, сложно версионировать, проблемы с масштабированием, отсутствие UI для конечного пользователя.
3. **Практика**:
   - Сборка агента, который ищет информацию в сети и сохраняет итог в Google Docs/Telegram.

#### Блок 4: True Coding подходы + Практика (1 час)
*Профессиональная разработка и контроль.*

1. **Зачем писать код?**
   - Полный контроль над промптами и логикой.
   - **Версионирование (Git)**: Безопасность изменений.
   - **Evals & Testing**: Как убедиться, что агент не поглупел (Unit-тесты для AI).
   - Экономика: Платим за токены, а не за выполнение операций на платформе.
2. **Стек технологий**:
   - Python: LangChain/LangGraph, PydanticAI.
   - JS/TS: Vercel AI SDK, LangGraph.js.
3. **Практика**:
   - Разбор структуры простого агента в коде.
   - Демонстрация: как запустить локально "Hello World" агента.
