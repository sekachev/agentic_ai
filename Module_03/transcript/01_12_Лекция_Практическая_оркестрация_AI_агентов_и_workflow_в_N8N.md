Тема
Основной вывод заключается в том, что для создания современных AI-агентов существует два основных подхода. Первый — использование визуальных low-code/no-code платформ, таких как n8n или iStudio, которые позволяют даже без глубоких навыков программирования строить сложные рабочие процессы (workflow), объединяя языковые модели, внешние API и память в наглядные схемы. Второй подход — традиционный кодинг (часто с помощью AI-ассистентов, "vibe-coding"), который предоставляет полную свободу и контроль, но требует технических навыков. Выбор между этими методами зависит от задачи: low-code платформы идеальны для быстрого прототипирования и типовых задач, в то время как кодинг необходим для сложных, кастомизированных и масштабируемых проектов.
Ключевые моменты
1.  Создание AI-агентов можно реализовать через low-code платформы (n8n, make.com, iStudio) или традиционный кодинг, который стал доступнее благодаря AI-ассистентам ("vibe-coding").
2.  Визуальные инструменты, такие как n8n, позволяют строить workflow, соединяя элементы (триггеры, модели, инструменты) в графической схеме, что значительно упрощает разработку.
3.  Ключевым элементом workflow является триггер — событие, запускающее процесс (например, сообщение в чате).
4.  Для работы с AI-моделями через API (например, OpenRouter) необходим API-ключ, который служит для авторизации запросов. Чувствительные данные, как ключи, следует хранить в отдельных файлах (например, .env).
5.  AI-агентам необходимо подключать инструменты (tools) — внешние API (например, для получения погоды), чтобы они могли получать доступ к актуальной информации, отсутствующей в их базовых знаниях.
6.  Платформы вроде n8n и iStudio позволяют интегрировать практически любой сервис, предоставляющий API (обычно в формате JSON), с помощью стандартных HTTP-запросов.
7.  Важным компонентом агента является память (memory), позволяющая сохранять контекст диалога.
8.  Любое AI-приложение состоит из backend (логика, LLM, инструменты) и frontend (пользовательский интерфейс). Визуальные конструкторы часто создают и то, и другое, но доставка такого приложения клиенту может быть затруднена.
9.  Несмотря на низкий порог входа, low-code платформы имеют недостатки: сложность отладки, проблемы с версионированием и зависимость от экосистемы (vendor lock-in).
10. "Vibe-coding" предоставляет полную свободу в реализации логики, доступ к огромному количеству open-source библиотек и простоту версионирования, а главным ограничением становится не технология, а фантазия разработчика.

Разделы
1.  Введение в инструменты для создания AI-агентов
  На рынке существуют различные платформы для создания AI-агентов, включая open-source решения (n8n), коммерческие сервисы (make.com) и визуальные конструкторы (iStudio). Эти инструменты позволяют гибко настраивать агентов, их промпты и подключаемые инструменты. Для тех, кто предпочитает код, существуют AI-ассистенты, упрощающие разработку ("vibe-coding").
2.  Настройка рабочей среды и создание Workflow
  Работа начинается с подключения провайдера языковых моделей (например, OpenRouter) с помощью API-ключа. Основой является создание Workflow на "холсте", где процесс запускается триггером (например, сообщение в чате). К агенту подключается чат-модель, компонент памяти для сохранения контекста и внешние инструменты.
3.  Интеграция внешних API в качестве инструментов
  Чтобы агент мог получать актуальную информацию (например, погоду), к нему подключаются инструменты. Это можно сделать через стандартные HTTP-запросы к открытым API (например, Open-Meteo). URL-запрос с необходимыми параметрами настраивается в соответствующем узле (например,   HTTP Request   в n8n) и описывается для модели, чтобы она знала, как его использовать.
4.  Сравнение подходов: Low-Code vs. Кодинг
  Low-code платформы (n8n, iStudio) предлагают быстрый старт, готовые шаблоны и визуальную разработку, но ограничены в гибкости, отладке и масштабировании. Кодинг ("vibe-coding"), наоборот, предоставляет полную свободу и мощность для сложных проектов и стал доступнее благодаря AI-ассистентам, снимающим высокий технологический барьер.
5.  Архитектура AI-приложения: Backend и Frontend
  Любое AI-приложение состоит из двух частей: backend (логика, LLM, инструменты) и frontend (пользовательский интерфейс). При кодовом подходе эти части создаются отдельно. Платформы вроде iStudio могут генерировать и то, и другое автоматически, но возникает проблема с доставкой готового продукта конечному пользователю.
Предложения
Разработчикам, привыкшим к классическому кодированию, важно перестроить мышление с категорий кода на категории задач для эффективной работы с AI-инструментами.
Вместо платных встроенных сервисов можно самостоятельно находить и интегрировать бесплатные внешние API для расширения функциональности агентов, используя узел HTTP Request.
В качестве домашнего задания предлагается создать простого Telegram-бота двумя способами: через low-code платформу (n8n) и с помощью кодинга, чтобы на практике сравнить время, усилия и результат.
Рекомендуется сосредоточиться на визуальных инструментах (NITEN, iStudio) для создания ботов, которые собирают и обрабатывают информацию, поскольку это более доступный путь для начинающих.
